# 백준 11726 🧱 2×n 타일링 파이썬

## 느낀점
>
- DP 규칙찾는데 시간을 너무 많이 썼다.
- 그냥 세어보니 피보나치

## 흐름

1. 2×n 직사각형을 1×2, 2×1 타일로 채우는 경우의 수
2. 점화식: `dp[n] = dp[n-1] + dp[n-2]`
   - 마지막에 2×1 타일 1개 세우는 경우: `dp[n-1]`
   - 마지막에 1×2 타일 2개 눕히는 경우: `dp[n-2]`
3. 피보나치 수열과 동일한 구조
4. 공간 최적화: 배열 대신 변수 2개로 계산

## 코드

```python
"""
1, 2, 3
"""
import sys
input = sys.stdin.readline
n = int(input())
result = 0
if n == 1:
    result = 1
elif n == 2:
    result = 2
else:
    i = 3
    prev_1 = 2
    prev_2 = 1
    while True:
        result = prev_1 + prev_2
        if(i == n):
            break
        else:
            prev_2 = prev_1
            prev_1 = result%10007
            i += 1
print(result%10007)
```

## 질문 & 실수 정리

### 질문했던 내용 (새로 배운 것)

| 질문 | 답변 |
|---|---|
| 왜 피보나치 점화식이 나오는지 모르겠음 | "마지막에 뭘 놓을 수 있는가"를 기준으로 경우를 나눈다. ① 2×1 세로 타일 → 나머지 dp[n-1], ② 1×2 가로 타일 2개 → 나머지 dp[n-2]. 이 두 가지가 유일한 선택지이므로 `dp[n] = dp[n-1] + dp[n-2]` |

### 실수했던 부분

| 틀린 것 | 올바른 것 |
|---|---|
| `prev_1 = i` (인덱스 대입) | `prev_1 = result` (계산 결과 대입) |
| 10007로 나눈 나머지 출력 누락 | `result % 10007` 출력 |
| `result` 변수 미초기화 (n=1,2일 때 런타임 에러) | `result = 0` 초기화 또는 분기 내에서 출력 |

## AI 코드 리뷰

### 개선할 점

1. **while True + break 패턴**: `for` 루프로 바꾸면 종료 조건이 명확해짐
2. **변수명**: `prev_1`, `prev_2`보다 `a`, `b`가 DP 문제에서 더 관용적
3. **초기화 로직**: `result = 0`은 의미 없는 값이라 어색함. 분기 구조를 정리하면 불필요

## 수정된 코드

```python
import sys
input = sys.stdin.readline

n = int(input())

if n == 1:
    print(1)
elif n == 2:
    print(2)
else:
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, (a + b) % 10007
    print(b)
```
